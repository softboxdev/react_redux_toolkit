
## Золотые правила 

1. **Начинайте с React useState/useContext** — не усложняйте, если не нужно
2. **Используйте Redux Toolkit** — никогда не пишите чистый Redux вручную
3. **Держите состояние минимальным** — храните только то, что нужно глобально
4. **Не кладите в Redux то, что можно вычислить** — вычисляйте в селекторах
5. **Выносите логику в слайсы** — один слайс = одна часть состояния

**Запомните:** Redux — это не панацея. Для простых приложений достаточно локального состояния React. Но когда приложение вырастает, Redux становится незаменимым инструментом для управления сложностью.
